# Security Policy

## Reporting a Vulnerability

Do not report security vulnerabilities in public GitHub Issues.

Use private reporting channels:
- GitHub Private Vulnerability Reporting (Security tab), if enabled
- Project maintainer email (if provided by repository owner)

Please include:
- affected component/service
- clear reproduction steps
- potential impact
- suggested mitigation (if known)

## Handling Policy

- Security reports are reviewed privately.
- Valid vulnerabilities are fixed and released as soon as practical.
- Public disclosure happens only after a fix or mitigation is available.
